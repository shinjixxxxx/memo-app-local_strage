# 同期機能コンセプト

メモアプリ（`memo-app_free-check-seet_02.html`）の同期機能の設計思想と各操作の意味をまとめる。

## 基本の考え方

- データはすべて各端末の **localStorage** に保存される（クラウドDBは持たない）。
- 端末間の共有は **JSONファイル1個**（iCloud Drive 等に置く）を介して行う。
- ファイルは上書きせず、書き出すたびに **日時付きの新ファイル**（`m-YYYY-MM-DD_HH-MM.json`）として保存する。どれが新しいか一目でわかり、誤操作からの復元もできる。

## 4つの操作の違い

各操作は「**読込**（ファイル→端末）」「**マージ**（両方を残す）」「**書出**（端末→ファイル）」の組み合わせで整理できる。

| 操作 | 読込 | マージ | 書出 | 用途 |
|---|:---:|:---:|:---:|---|
| **同期する** | ○ | ○ | ○ | 双方向同期。読み込んでマージし、書き戻す |
| **マージ読込** | ○ | ○ | ✗ | ファイルの内容を取り込むだけ（ファイルは変更しない） |
| **強制読込** | ○ | ✗（置換） | ✗ | ファイルを正として端末を丸ごと上書き |
| **強制書出** | ✗ | ✗ | ○ | 端末を正としてファイルを丸ごと上書き |

### 同期する（双方向・マージ）
ファイルを読み込み、端末のデータとマージ（和集合・全部残す）し、結果を新ファイルへ書き出す。通常の同期はこれを使う。

### マージ読込（一方向・マージ・書き戻さない）
ファイルを選んで読み込み、端末とマージする。**ファイルへは書き戻さない**。「他端末のメモを取り込みたいが、ファイル側はそのままにしたい」場合に使う。

### 強制読込（一方向・置換）
選んだファイルの内容で端末を**完全に置換**する。端末だけにあるメモは消える。「ファイルを正にして端末をリセットしたい」場合に使う。マージは行わない。

### 強制書出（一方向・置換）
端末の内容で**新ファイルを書き出す**。ファイル側にしかないメモ（他端末の未取り込み分）は引き継がれない。「この端末を正にしたい」「削除をファイルに反映したい」場合に使う。

## マージのルール（和集合）

「同期する」「マージ読込」で行われるマージは、**両方のメモを残す（消さない）** ことを原則とする。

- **行メモ**: メモ文字列をキーに重複排除し、両方の行を残す。チェックボックスは論理和（どちらかがONならON）。
- **末尾の長文メモ（メモ1/メモ2）**: 行単位の和集合（重複行を除いて両方を残す）。
- **ページ**: ページIDを基準に統合。両方にあるページはマージ、片方だけのページはそのまま残す。
- **日付**: 新しい方（`yyyy-mm-dd` の辞書順で大きい方）を採用。

削除を全端末に反映したい場合は、マージでは消えないため **強制書出／強制読込** を使う（1台で強制書出 → 他端末で強制読込）。

## ファイル名の方式

- 書き出しは常に日時スタンプ付き: `m-YYYY-MM-DD_HH-MM.json`（例: `m-2026-06-08_14-30.json`）。
- 上書きせず毎回新規ファイルとして残すため、世代管理・復元が容易。
- 溜まったファイルは手動で整理する運用。

## 端末別の入出力経路

| 端末 | 書き出し | 読み込み |
|---|---|---|
| Mac（Chrome/Edge） | File System Access API でファイル保存ダイアログ | ファイル選択ダイアログ |
| iPhone/iPad（Safari） | Web Share API の共有ダイアログ（iCloud Drive 等へ保存先を選択） | ファイル選択 |

### iPhone での注意（HTTPS 必須）
Web Share API は **セキュアコンテキスト（HTTPS）でしか動作しない**。LAN の HTTP 接続では共有ダイアログが出ず、ブラウザ既定のダウンロードフォルダに固定保存される。そのため開発サーバ（`server.js`）を自己署名証明書で HTTPS 化し、iPhone に証明書を信頼させて HTTPS でアクセスする必要がある。

## 自動同期

- 設定ダイアログのトグルで ON/OFF（**デフォルト OFF**）。
- ON のときは指定間隔（分）で自動的に「同期する」を実行する。Mac かつ無操作で書き込み許可がある場合のみ動作する（権限プロンプトは出さない）。
